Tratamento de Exceções em Python

Durante a execução de um programa, é comum que erros inesperados aconteçam, como dividir um número por zero ou tentar acessar um índice inexistente em uma lista. Esses erros são chamados de exceções, e o Python possui mecanismos para lidar com eles de forma elegante e segura.

O tratamento de exceções permite que o programa continue funcionando mesmo quando um erro ocorre, evitando que ele seja encerrado de forma abrupta. Isso é essencial para criar aplicações robustas e confiáveis.

Compreendendo uma Exceção

Uma exceção é uma situação fora do comum que ocorre durante a execução do programa. Quando o Python encontra um erro, ele gera uma exceção. Se ela não for tratada, o programa é encerrado com uma mensagem de erro.

Veja um exemplo simples onde ocorre uma exceção:

numero = int(input("Digite um número: "))
resultado = 10 / numero
print("Resultado:", resultado)

Se o usuário digitar 0, o programa tentará dividir por zero e causará uma exceção do tipo ZeroDivisionError.

Usando o try/except

Para evitar que o programa seja interrompido com um erro, usamos a estrutura try/except. O código dentro do bloco try é executado normalmente. Se uma exceção ocorrer, o Python pula para o bloco except.

try:
    numero = int(input("Digite um número: "))
    resultado = 10 / numero
    print("Resultado:", resultado)
except ZeroDivisionError:
    print("Erro: divisão por zero não é permitida.")

Agora, se o usuário digitar 0, o programa não será interrompido, e uma mensagem amigável será exibida.

Tratando Múltiplas Exceções

Podemos tratar diferentes tipos de erros com blocos except específicos, ou com um bloco genérico para qualquer exceção.

try:
    valor = int(input("Digite um número inteiro: "))
    print("Você digitou:", valor)
except ValueError:
    print("Erro: você deve digitar um número inteiro.")
except Exception as erro:
    print("Ocorreu um erro inesperado:", erro)

O primeiro except trata especificamente o erro de conversão. Já o segundo pega qualquer outro tipo de erro que possa ocorrer.

Usando else e finally

Podemos complementar o bloco try com else e finally:

  • else: é executado se nenhum erro ocorrer;
  • finally: sempre é executado, com ou sem erro.
try:
    num = int(input("Digite um número positivo: "))
    if num < 0:
        raise ValueError("Número negativo")
except ValueError as e:
    print("Erro:", e)
else:
    print("Número aceito!")
finally:
    print("Execução finalizada.")

O bloco finally é útil, por exemplo, para fechar arquivos ou conexões, mesmo que ocorra um erro no programa.

Lançando Exceções com raise

Em algumas situações, podemos querer forçar um erro para interromper o fluxo normal do programa. Para isso, usamos o comando raise.

idade = int(input("Digite sua idade: "))
if idade < 0:
    raise ValueError("Idade não pode ser negativa!")
print("Idade registrada com sucesso.")

O raise permite lançar manualmente uma exceção, útil para impor regras no programa.

Conclusão

O tratamento de exceções é um recurso essencial da linguagem Python. Com ele, podemos tornar nossos programas mais seguros, evitando que pequenos erros interrompam grandes processos. Use sempre que trabalhar com entradas do usuário, arquivos, conexões ou operações críticas.
